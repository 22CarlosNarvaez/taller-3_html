Git + VS Code

Paso a paso para conectar Git con la terminal de VS Code y subir tu proyecto a GitHub.

Requisitos previos
Tener instalados Git y VS Code, y tener una cuenta en GitHub.
1

Verificar la instalación de Git

Abre la terminal de VS Code con Ctrl + ` y escribe:

git --version
Si ves algo como git version 2.x.x, Git está instalado correctamente.
2

Configurar tu identidad en Git

Solo necesitas hacer esto una vez. Usa el mismo correo de tu cuenta de GitHub:

git config --global user.name "Tu Nombre"
git config --global user.email "tucorreo@ejemplo.com"

Para verificar que quedó guardado:

git config --list
3

Inicializar un repositorio local

Navega hasta la carpeta de tu proyecto y ejecuta:

cd ruta/de/tu/proyecto
git init
Esto crea una carpeta oculta .git en tu proyecto. Ahí Git guarda todo el historial.
4

Agregar archivos y hacer el primer commit

Primero agrega todos los archivos al área de preparación (staging):

git add .

Luego confirma los cambios con un mensaje descriptivo:

git commit -m "primer commit"
git add . agrega todos los archivos. Usa git add archivo.html para agregar uno solo.
El mensaje del commit debe describir qué cambios hiciste. Ej: "agrega sección contacto".
5

Crear un repositorio en GitHub

  1. Ve a github.com/new.
  2. Escribe el nombre del repositorio (ej: devfolio).
  3. Elige si será Público o Privado.
  4. No marques la opción de inicializar con README (ya tienes archivos locales).
  5. Haz clic en Create repository.
GitHub te mostrará una URL del repositorio. Cópiala, la necesitarás en el siguiente paso.
6

Conectar tu repo local con GitHub

En la terminal de VS Code, ejecuta los siguientes comandos (reemplaza la URL con la de tu repositorio):

git branch -M main
git remote add origin https://github.com/tu-usuario/tu-repo.git
git push -u origin main
La primera vez que hagas push, VS Code o el navegador te pedirá autenticarte con tu cuenta de GitHub. Sigue las instrucciones en pantalla.

Flujo de trabajo diario

Cada vez que hagas cambios en tu proyecto, repite estos tres pasos:

1. Agregar
git add .

Prepara los archivos modificados.

2. Confirmar
git commit -m "mensaje"

Guarda los cambios con un mensaje.

3. Subir
git push

Sube los cambios a GitHub.

Comandos útiles

Comando ¿Para qué sirve?
git status Ver qué archivos tienen cambios pendientes.
git log --oneline Ver el historial de commits de forma resumida.
git pull Traer los cambios más recientes desde GitHub.
git branch nombre-rama Crear una nueva rama para trabajar sin afectar main.
git checkout nombre-rama Cambiarse a otra rama.
git merge nombre-rama Unir los cambios de una rama a la actual.
git clone URL Descargar un repositorio de GitHub a tu computador.
